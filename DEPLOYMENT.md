## Complete Deployment Process

### Phase 1: Pre-Deployment (10 min)

#### Step 1: Verify Environment
```bash
# Check Python version (need 3.11+)
python --version

# Verify project structure
python verify_structure.py

# Run pre-flight checklist
python deployment_checklist.py
```

#### Step 2: Run Setup Wizard
```bash
# Interactive setup (recommended for first-time)
python setup.py

# This will:
# - Create all directories
# - Install dependencies
# - Configure .env file
# - Test API connection
```

### Phase 2: Build for Production (5 min)

```bash
# Generate production artifacts
python build_for_production.py

# This creates:
# - DEPLOYMENT_GUIDE.md
# - config/production_config.py
# - Dockerfile
# - docker-compose.yml
# - .dockerignore
```

### Phase 3: Testing (5 min)

```bash
# Test individual steps
python scripts/step5_extract_observations.py  # Extract
python scripts/step6_merge_and_conflict.py   # Merge
python scripts/step7_severity_scoring.py     # Score
python scripts/step8_generate_ddr.py         # Report

# Or test complete pipeline
python main.py

# Or use production main
python prod_main.py
```

### Phase 4: Deployment (Varies)

#### Option A: Local/Direct Deployment
```bash
# Simple execution
python main.py

# With logging
python prod_main.py
```

#### Option B: Docker Deployment
```bash
# Build Docker image
docker build -t ai-ddr-generator .

# Run container
docker run -v $(pwd)/data:/app/data \
           -v $(pwd)/outputs:/app/outputs \
           -e GEMINI_API_KEY=your_key \
           ai-ddr-generator

# Or use docker-compose
docker-compose up
```

#### Option C: Scheduled (Linux/macOS)
```bash
# Create systemd service
sudo cp ai-ddr-generator.service /etc/systemd/system/

# Enable and start
sudo systemctl enable ai-ddr-generator
sudo systemctl start ai-ddr-generator
```

#### Option D: Windows Scheduled Task
```bash
# Run batch file
deployment.bat

# Or schedule with Task Scheduler
# See: deployment.bat
```

## Configuration for Deployment

### 1. Environment Variables (.env)
```bash
# Required
GEMINI_API_KEY=your_actual_key_here

# Optional
LOG_LEVEL=INFO           # DEBUG, INFO, WARNING, ERROR
ENVIRONMENT=production   # production, staging, development
PDF_TIMEOUT=60          # Seconds
API_MAX_RETRIES=3       # Retry attempts
BATCH_SIZE=10           # Items per batch
```

### 2. Application Settings (config/app_config.py)
```python
# Modify as needed for your deployment
FUZZY_MATCH_THRESHOLD = 70  # Merge accuracy
TIMEOUT_SECONDS = 30        # API timeout
LOG_LEVEL = "INFO"          # Logging level
ENABLE_GEMINI_FALLBACK = True  # Offline fallback
```

### 3. Production Settings (config/production_config.py)
```python
# Auto-generated by build_for_production.py
ENVIRONMENT = "production"
DEBUG = False
LOG_LEVEL = "WARNING"
API_ENDPOINT = "https://..."
BATCH_SIZE = 10
```

## Deployment Architectures

### Single Machine
```
Developer/Test Machine
    ↓
Python 3.11+ → main.py → outputs/
    ↓
Reports ready for use
```

### Docker Container
```
Docker Registry
    ↓
Docker Build (Dockerfile)
    ↓
Container Execution (docker-compose)
    ↓
Mapped volumes → outputs/
```

### Linux Server with Systemd
```
Git Repository
    ↓
Clone to /opt/ai-ddr-generator
    ↓
systemd service (ai-ddr-generator.service)
    ↓
Scheduled execution (timer.service)
    ↓
Outputs → /opt/ai-ddr-generator/outputs/
```

### Windows Server
```
Git Repository
    ↓
Clone to C:\Projects\ai-ddr-generator
    ↓
Windows Task Scheduler + deployment.bat
    ↓
Outputs → C:\Projects\ai-ddr-generator\outputs\
```

## Troubleshooting Deployment

### Issue: "API key not found"
```bash
# Check .env file exists
cat .env  # May vary by OS

# Regenerate .env
rm .env
python setup.py
```

### Issue: "Module not found"
```bash
# Reinstall dependencies
pip install --upgrade pip
pip install -r requirements.txt

# Or use virtual environment
python -m venv venv
source venv/bin/activate  # or venv\Scripts\activate on Windows
pip install -r requirements.txt
```

### Issue: "Port already in use (Docker)"
```bash
# Find service using port
lsof -i :8000          # Linux/Mac
netstat -ano | findstr :8000  # Windows

# Stop existing service
docker stop container_name

# Or use different port
docker-compose -f docker-compose.override.yml up
```

### Issue: "Permission denied" (Linux/Mac)
```bash
# Make scripts executable
chmod +x setup.py
chmod +x main.py
chmod +x deployment_checklist.py
chmod +x build_for_production.py
```

### Issue: "Gemini API quota exceeded"
**NO ACTION NEEDED** — System automatically falls back to template-based report generation

## Deployment Maintenance

### Daily
- Monitor `app.log` for errors
- Check `outputs/` for recent reports
- Verify outputs are as expected

### Weekly
- Review API usage (Google AI Studio)
- Check for new Python packages/security updates
- Test full pipeline: `python main.py`

### Monthly
- Rotate API keys
- Update dependencies: `pip install -r requirements.txt --upgrade`
- Review logs for patterns
- Archive old outputs if needed

## Rollback Procedure

If deployment has issues:

```bash
# 1. Stop current deployment
docker-compose down        # If using Docker
systemctl stop ai-ddr-generator  # If using systemd
killall python            # If direct execution

# 2. Revert to last working version
git checkout last_working_commit

# 3. Reinstall dependencies
pip install -r requirements.txt

# 4. Redeploy
python setup.py
python main.py
```
